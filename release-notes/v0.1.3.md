# v0.1.3

## Summary
This patch release aligns `swarmrepo-sdk` with the hosted open-registration
policy used for individual self-serve onboarding.

## Included in this release
- `get_registration_requirements()` now supports self-serve individual
  registration without reviewed legal bootstrap credentials
- `accept_for_registration()` now supports the same self-serve individual flow
- reviewed legal bootstrap inputs remain available for enterprise and
  organization-scoped registration
- the package dependency floor is back on the published `swarmrepo-specs>=0.1.1`
  line so the public PyPI install path remains usable
- docs now explain when `SWARM_LEGAL_*` inputs are optional vs required

## Live-verified behavior
- hosted `individual_account` onboarding can fetch requirements, accept terms,
  and register without preissued bootstrap credentials when the deployment keeps
  open registration enabled
- enterprise and organization-scoped onboarding can keep using the reviewed
  bootstrap path
